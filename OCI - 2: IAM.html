Identity & Access Management

Video 3: IAM Introduction

The Two Pillars: AuthN vs. AuthZ

Authentication (AuthN): Who are you? Verifying identity (e.g., username/password).

Authorization (AuthZ): What can you do? Granting permissions (e.g., allowing a user to create VMs).

Core IAM Components

Identity Domain: A container for users and groups; the master directory.

Principals (The "Who"): Users and Instance Principals (permissions for Compute instances).

Groups: Collections of users needing the same access.

Compartments: Logical folders to organize and isolate resources.

OCID (Oracle Cloud ID): A unique, Oracle-assigned ID for every single cloud resource.

Video 5: IAM Authentication & Authorization

Authentication Methods

Username & Password: For interactive console sessions.

API Signing Keys: For programmatic access (CLI, SDKs).

Auth Tokens: For third-party APIs that don't support OCI's native authentication.

Authorization with Policies

Core Principle: Deny by default. You must explicitly allow permissions.

Syntax: `Allow group to in `

Verbs (Least to Most Privilege): `inspect` -> `read` -> `use` -> `manage`

Video 7: Tenancy Setup Best Practices

The Three Foundational Rules

  1. Delegate Administration: Don't use the root admin account for daily tasks. Create a dedicated admin group.
  2. Use Dedicated Compartments: Isolate resources logically. Don't use the root compartment.
  3. Enforce Multi-Factor Authentication (MFA): Add an extra layer of security for all users.
